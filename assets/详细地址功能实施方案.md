# 唐小识小程序 · 详细地址功能实施方案

> 目标：实现淘宝/美团式「地图选点 + 手填门牌号/联系人/电话 + 地址簿」
> 约束：地图选点用微信原生 `Taro.chooseLocation`；数据暂存内存/本地缓存，先跑通功能，上线前再接数据库。
> 环境：扣子编程 · Taro 4 + React + NestJS。本文档只给方案与结构，不含成品代码。

---

## 0. 项目现状速览（施工前必读）

| 层 | 现状 | 对地址功能的影响 |
|---|---|---|
| 首页 `src/pages/index/index.tsx` | 发/收货地址已用 `chooseLocation` 拿到 POI，但只存了 `name/address/经纬度` | 需扩展为详细地址，并接入地址簿 |
| `src/utils/amap.ts` | 有 `chooseLocation()`、`AddressInfo` 接口 | 在此扩展字段与工具函数 |
| 表单输入 | 门牌号/联系人/电话等目前是假的 `<View><Text>占位</Text>`，**没有真 Input** | 必须换成真实 `Input/Textarea` 才能填字 |
| 后端 `server/src/orders` | 内存数组存储 | 地址簿同样先用内存 Service，后续接 Drizzle |
| 登录 | 无微信登录、无 openid | 地址簿暂时按「本地设备」维度存；接登录后再按 openid 归属 |

## 1. 数据结构设计

### 1.1 前端 AddressInfo 扩展（`src/utils/amap.ts`）

在现有 `AddressInfo` 基础上新增字段（全部可选，兼容旧数据）：

| 字段 | 类型 | 来源 | 说明 |
|---|---|---|---|
| `id` | string | 本地生成 | 地址簿唯一标识 |
| `name` | string | chooseLocation | POI 名称（如"XX大厦"）·已有 |
| `address` | string | chooseLocation | 省市区街道详细地址·已有 |
| `latitude` / `longitude` | number | chooseLocation | 经纬度·已有 |
| `province` / `city` / `district` | string | chooseLocation | 省市区·已有（部分） |
| `houseNumber` | string | **手填 Input** | 门牌号/楼层/房间，如"3号楼2单元801" |
| `contactName` | string | **手填 Input** | 联系人姓名 |
| `contactPhone` | string | **手填 Input** | 联系电话 |
| `tag` | '家' \| '公司' \| '仓库' \| '其他' | 选择 | 地址标签 |
| `isDefault` | boolean | 开关 | 是否默认地址 |
| `role` | 'pickup' \| 'delivery' \| 'both' | 可选 | 发货/收货用途，便于分类 |

> 说明：`chooseLocation` 只能拿到"地图上的点 + POI 名"，**门牌号、联系人、电话必须靠用户手填**——这正是美团/淘宝"两步式"的关键。

### 1.2 完整显示地址拼接规则

展示时按此顺序拼：`address（省市区街道） + houseNumber（门牌号）`，
列表卡片主标题用 `contactName + 尾号phone`，副标题用完整地址。参考美团样式。

### 1.3 后端类型（`server/src/address/address.types.ts`，新建）

与前端 `AddressInfo` 字段对齐，额外加：
- `id`、`ownerKey`（暂用前端传的设备标识/临时 id，接登录后换 openid）、`createdAt`、`updatedAt`。

## 2. 页面与文件规划

### 2.1 新增页面

| 页面路径 | 文件 | 职责 |
|---|---|---|
| `pages/address/list` | `index.tsx` / `index.config.ts` / `index.css` | 地址簿列表：展示、新增入口、编辑、删除、设默认、选中回填 |
| `pages/address/edit` | 同上三件套 | 新增/编辑单条地址：地图选点 + 门牌号/联系人/电话 Input 表单 + 标签 + 默认开关 |

> 建议放进**分包**（`subPackages`）以控制主包体积（见 DESIGN.md 包体积约束）。

### 2.2 需要在 `src/app.config.ts` 注册

- 把两个地址页加入 `pages` 或 `subPackages`。
- 新增权限声明（chooseLocation 必需）：
  - `permission.scope.userLocation`（desc 文案）
  - `requiredPrivateInfos: ['chooseLocation', 'getLocation']`
  - ⚠️ 这两项不配，`chooseLocation` 在真机/审核会失败。

### 2.3 新增状态管理（`src/stores/address.ts`，zustand）

职责：
- `addressList`：地址簿数组（内存 + 同步到 `Taro.setStorageSync` 本地缓存，冷启动恢复）
- `addAddress / updateAddress / removeAddress / setDefault`
- `getDefault()`、按 role 过滤
- 选中态：供下单页选地址后回填

> 先做纯前端本地缓存版即可跑通；后端 address 模块作为可选同步层。

## 3. 核心交互流程（两步式，对齐美团）

### 3.1 新增/编辑地址（`pages/address/edit`）

```
进入编辑页
  ├─ 点「定位到地图选点」→ Taro.chooseLocation()
  │     └─ 回填：name、address、province/city/district、经纬度
  ├─ 手填 Input：门牌号(houseNumber)  ← 真实 Input，必填
  ├─ 手填 Input：联系人(contactName)  ← 真实 Input，必填
  ├─ 手填 Input：联系电话(contactPhone) ← 真实 Input，必填 + 手机号校验
  ├─ 选择标签 tag（家/公司/仓库）    ← 可用 ToggleGroup 或 Badge 点选
  ├─ 开关 isDefault（设为默认）       ← Switch
  └─ 点「保存」→ 校验 → store.addAddress/updateAddress → 返回上一页
```

校验规则：
- 未选点（无经纬度）→ 提示"请先在地图选择位置"
- houseNumber / contactName 为空 → 提示
- contactPhone 不符合 `/^1[3-9]\d{9}$/` → 提示"手机号格式不正确"

### 3.2 地址簿列表（`pages/address/list`）

- 列表项：联系人 + 电话尾号 · 标签徽标 · 完整地址 · [默认]角标
- 操作：点整项 = 选中并回填到下单页；右侧「编辑」进 edit 页；「删除」二次确认（用 `AlertDialog`）
- 顶部/底部：「+ 新增地址」按钮 → 进 edit 页
- 空状态：`Skeleton` 或空占位 + 引导新增
- 可选：`Taro.chooseAddress` 一键导入微信收货地址（需后台开通该接口权限）

### 3.3 下单页改造（`src/pages/index/index.tsx`）

现状：`handleChoosePickup/handleChooseDelivery` 直接调 `chooseLocation`。
改为：
- 点发/收货地址 → `Taro.navigateTo` 到 `pages/address/list?role=pickup`（带用途参数）
- 列表选中后回填（通过 store 或 `Taro.eventCenter` / 路由返回参数传回）
- 地址卡片展示升级：联系人 + 电话 + 完整地址（含门牌号），而非只有 POI 名
- 计价里程仍用回填地址的经纬度（`calculateDrivingDistance` 逻辑不变）

## 4. 后端方案（内存版，`server/src/address/`，新建）

仿照现有 `orders` 模块结构：

| 文件 | 职责 |
|---|---|
| `address.module.ts` | 注册模块，import 进 `app.module.ts` |
| `address.types.ts` | 地址类型（对齐前端 + ownerKey/时间戳） |
| `address.service.ts` | 内存数组 CRUD：`create/findAll(ownerKey)/findOne/update/remove/setDefault` |
| `address.controller.ts` | 路由（注意：路径不带 `api` 前缀，成功统一 200） |

接口清单（前缀 `/api` 自动加）：
- `POST /address` 新增
- `GET  /address?ownerKey=xxx` 列表
- `GET  /address/:id` 详情
- `POST /address/:id`（或 PUT）更新
- `POST /address/:id/delete` 删除
- `POST /address/:id/default` 设默认

> 现阶段前端可**只用本地缓存**、不依赖后端也能跑通；后端接口作为"上线前接数据库"的过渡骨架先搭好。

## 5. 跨端 / 小程序注意事项（照 AGENTS.md）

1. **必须用真 Input**：门牌号/联系人/电话务必用 `@/components/ui/input`，且 H5 端要 `View` 包裹、样式放外层（AGENTS.md「H5/小程序兼容」）。
2. **手机号输入**：`Input` 加 `type="number"` / `maxlength={11}`。
3. **样式用 Tailwind**：禁止 `w-[340px]`、`style` 硬编码 px。
4. **chooseLocation 前置条件**：`app.config.ts` 配好 permission + requiredPrivateInfos；微信公众平台后台需在「开发管理-接口设置」勾选位置接口。
5. **底部固定按钮**避开 TabBar：`bottom: 50`+，用 inline style（Fixed+Flex 在 H5 失效）。
6. **删除确认**用 `AlertDialog`，不要手搓。
